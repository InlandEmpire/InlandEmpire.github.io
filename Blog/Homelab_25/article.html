V41H4114 - Components of a Homelab

Foreword

Since starting just over a year ago, homelabbing has become my number one technical hobby. While my expectations were low, having only conceptualized my server first as a network-attached storage device, I quickly learned how much a dedicated home server is capable of. As my ambitions grew, so did my technical knowledge, which skyrocketed to include robust understandings of network infrastructure, computer hardware, and distributed systems. Unfortunately, since I knew so little starting out, I made a number of mistakes in choosing hardware. Had I been exposed to more information beforehand, I would have made very different component choices in my build.To hopefully help someone down the road, I've compiled this list of each of my components where I explain the logic supporting their inclusion.

Use cases

My homelab serves a number of simultaneous roles. The current build is optimized for the following:
• High capacity network-attached storage
• Parallel virtualization
• 24/7 Seeding
• DNS Sink Ad Blocking
• Internal Wiki hosting
• Media Streaming (Jellyfin to Infuse)
• Bare-metal performance VM gaming

Components

Case

Fractal Design Node 804

When I first built my rig, I didn't have space to hide a server rack in my home. Because of that, aesthetics and presentability were important. I wanted something small and inconspicuous that could reside anywhere from a living room to a home office. Fortunately, Swedish case manufacturer Fractal Design created the Node 804— a Micro ATX NAS/homelab case. It's bifurcated design supports a whopping eight 3.5” drives, an additional four 2.5” drives and one 5.25” optical drive, all in an elegant cube form factor. The Node line is a favorite among homelabbers, and it was the perfect choice for me. As I love the cyberpunk style, I've added dozens of tech-themed stickers to the front and side panels from Defcon and other events.
